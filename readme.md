# 오빠 오늘 무슨 날인지 몰라?

공개 달력에 기념일을 등록하고 함께 보는 **기념일 기반 소셜 캘린더 서비스**입니다.  
연인, 부부, 가족, 친구 사이의 생일이나 각종 기념일을 공개적으로 기록하고 구경할 수 있는 가벼운 재미 중심의 서비스로 시작합니다.

\---

## 1\. 프로젝트 한 줄 소개

**“오빠 오늘 무슨 날인지 몰라?”** 는  
누구나 공개 달력에서 기념일을 구경할 수 있고, 로그인한 사용자는 자신의 기념일 3개를 무료로 등록할 수 있는 서비스입니다.

핵심은 단순한 개인 일정 관리가 아니라,  
**공개된 기념일을 함께 보고 즐기는 소셜형 달력 경험**입니다.

\---

## 2\. MVP 목표

이번 MVP의 목표는 아래 3가지를 검증하는 것입니다.

1. **공개 달력 구조 자체가 재미있는가**
2. **사용자가 자신의 기념일을 직접 등록하고 싶어하는가**
3. **로그인 없이 구경 → 로그인 후 등록으로 이어지는 흐름이 자연스러운가**

즉, 처음부터 기능을 많이 넣기보다  
**공개 달력 + 기념일 등록 + 프로필 관리**까지의 최소 구조를 빠르게 구현하는 것이 목표입니다.

\---

## 3\. MVP 범위

### 포함 기능

#### 1\) 공개 메인 달력

* 로그인하지 않아도 접근 가능
* 월간 달력을 크게 표시
* 각 날짜에 등록된 기념일 개수 표시 > UI상 기념일을 등록한 순서나 반응(참여자 등)이 좋은 순서로 노출하고 나머지는 "+0개" 등으로 표시, 일반 캘린더 UI참고
* 날짜 클릭 시 해당 날짜의 기념일 목록 확인 가능

#### 2\) Google 간편 로그인

* Google OAuth 로그인 지원
* 로그인 후 사용자 계정 생성

#### 3\) 기념일 등록

* 로그인한 사용자는 **기념일 3개를 무료로 등록**
* 등록 가능한 정보

  * 기념일 제목
  * 날짜
  * 카테고리
  * 설명
  * 반복 여부(매년 반복 / 1회성)

#### 4\) 기념일 목록 조회

* 날짜 클릭 시 해당 날짜에 등록된 기념일 목록 표시
* 목록에서 확인 가능한 정보 예시

  * 기념일 제목
  * 등록자 닉네임
  * 카테고리

#### 5\) 마이페이지

* 내 프로필 확인 및 수정
* 수정 가능한 정보

  * 닉네임
  * 프로필 사진
* 내가 등록한 기념일 확인
* 내 기념일 수정 / 삭제 가능

\---

## 4\. MVP 제외 범위

아래 기능은 이번 MVP에서는 제외합니다.

* 파티 / 모임 생성
* 참석 여부 체크
* 참석자 목록 공개
* 영향력 지표 / 랭킹
* 알림 기능
* 유료 플랜
* 3개 초과 기념일 등록
* 고급 검색 / 필터
* 소셜 공유 최적화

이유는 명확합니다.  
우선은 **공개 달력 경험 자체가 살아있는지**를 먼저 확인해야 하기 때문입니다.

\---

## 5\. 핵심 사용자 흐름

### 비로그인 사용자

1. 메인 페이지 진입
2. 공개 달력 확인
3. 특정 날짜 클릭
4. 해당 날짜의 기념일 목록 확인
5. 서비스 재미를 느끼고 로그인 유도

### 로그인 사용자

1. Google 로그인
2. 닉네임 / 프로필 설정
3. 기념일 3개 등록
4. 내 기념일 관리
5. 공개 달력에서 내 기념일 노출

## 8\. 기본 정책

### 공개 정책

* 공개 달력과 공개 기념일 목록은 비로그인 사용자도 열람 가능
* 개인정보는 최소화
* 실명 대신 닉네임 중심 노출

### 등록 정책

* 사용자당 무료 등록은 3개
* 본인이 등록한 기념일만 수정 / 삭제 가능

### 프로필 정책

* 닉네임은 공개 노출
* 프로필 사진은 선택적으로 사용

## 9\. 제품 톤앤매너

이 서비스는 완전히 진지한 일정 관리 도구라기보다,  
**밈 요소와 실사용 가능성이 섞인 반실용 서비스**를 지향합니다.

톤은 아래처럼 가져갑니다.

* 첫인상은 재밌고 가볍게
* 사용 흐름은 직관적이고 단순하게
* 데이터 노출은 공개형이지만 과도한 개인정보는 배제
* “웃긴 서비스”로 시작하되, 실제로 등록하고 싶을 정도의 완성도는 확보

## 14\. 프로젝트 요약

이 MVP는  
**“기념일 저장 서비스”가 아니라 “공개적으로 기념일을 구경하고 등록하는 놀이형 소셜 달력”** 을 만드는 것이 핵심입니다.

처음부터 큰 서비스를 만들기보다,  
짧은 기간 안에 빠르게 구현해서 사용자 반응을 확인하는 데 집중합니다.
